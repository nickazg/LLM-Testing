6. Claude Code vs Kilo

Kilo massively outperforms Claude Code for most budget models. gemma-4-27b: CC 7% vs Kilo 100% (+93pp). Devstral: CC 13% vs Kilo 96% (+83pp). gemma-4-31b: CC 25% vs Kilo 67% (+42pp). CC's heavier system prompt and tool definitions consume more context budget, devastating models with smaller effective context windows. Exception: GLM-5 performs better on CC (+5%) — powerful enough to benefit from richer tool descriptions.

8. Why Models Performed As They Did

Failure Mode Analysis

Why Gemma underperforms vs online benchmarks

  • CLI scaffolding overhead: Models must operate within Claude Code/Kilo's tool-use framework. This consumes context and requires understanding JSON tool schemas.
  • OpenRouter routing: Adds latency, sometimes truncates prompts.
  • Timeout pressure: gemma-4-31b averages 223s/run — many T3 tasks time out at 300s even when the model "knows" the answer.
  • Import errors: gemma-4-27b fails with import/module errors on 86% of failed runs — the model generates code referencing nonexistent modules.
  • VFX domain gap: Online benchmarks test general coding. Our T3/T4 tasks test niche USD/Houdini APIs unlikely to be in gemma's training data.

The benchmark measures "can this model do real work through a CLI tool?" — not "can this model answer coding questions?"

Score Distribution: Binary, Not Gradual

Correctness scores are almost always 1.00 or 0.00. Very few partial scores exist. Models either fully understand a task or fail completely — there is no "partially correct" middle ground.

9. Recommendations

Use

  • GLM-5 (87%) — Budget VFX/pipeline work. Use Z.ai coding API.
  • glm-4.5-air-free (72%) — Best free option. Surprisingly strong.
  • qwen3-30b (67%) — Solid all-rounder for general coding.
  • Workflow/light skills — For known CS problems, concise guardrails beat prescriptive recipes.
  • Context/heavy skills — For org conventions, more detail = better compliance.

Avoid

  • gpt-oss-120b/20b (2%) — Non-functional through CLIs.
  • minimax-m2.7 (3%) — Non-functional.
  • devstral + skills — 0% with any skill injected. Use only without skills.
  • DSPy compilation on weak models — Zero or negative effect below capability threshold.
  • Heavy workflow skills — Prescriptive recipes conflict with model's internal knowledge.
